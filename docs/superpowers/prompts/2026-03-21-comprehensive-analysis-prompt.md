# Comprehensive Narrative Data Analysis: Full Terrain Pass

## Who You Are

You are analyzing 7.1MB of narrative data generated through a data-driven elicitation process for a storytelling engine called "storyteller." The data was generated by asking a 35B parameter model (qwen3.5:35b) to analyze 30 genre regions across 12 primitive types. The data is rich, qualitative, analytically insightful, and occasionally poetic. Your job is to find the structural patterns that will inform how this data is persisted, queried, and used at runtime.

You are operating in the `storyteller-data` repository. All data lives under `narrative-data/`.

## What the Data Is

### Genre Regions (30 genres, ~449KB)
`narrative-data/genres/{slug}/region.raw.md`

Each genre is described as a position in ~30-dimensional narrative space: aesthetic (sensory density, groundedness, prose register), tonal (dread↔wonder, cynicism↔earnestness, irony), temporal (linear↔cyclical, pacing), thematic (power, identity, knowledge, connection as weighted tag sets), agency (determinism↔agency, collective↔individual), locus of power (ranked: Place, Person, System, Relationship, Cosmos), world affordances (magic, technology, violence, death, supernatural), epistemological stance (knowability, knowledge reward, narration reliability).

### Discovery Primitives (Phase 1 per-genre + Phase 2 cluster synthesis)
Each type has 30 per-genre extraction files and 6 cluster synthesis files.

## What We Need From You

### 1. Axis Inventory Update

The first analysis identified ~30 universal genre dimensions and 8 canonical state variables. With the full corpus, update this:

- **New axes or dimensions** that the dynamics, goals, spatial topology, and place-entity data surfaced. Are there dimensions the first pass missed?
- **Axis interactions** that only become visible with the full data. The dynamics data shows how axes relate to each other at different scales (orbital/arc/scene). The spatial topology data shows how axes interact across space. Are there interaction patterns that should be modeled as constraints or as composite dimensions?
- **State variable evolution** — the goals and dynamics data shows how state variables change at different temporal scales. Are the 8 canonical state variables still sufficient, or does the full data reveal additional canonical variables?
- **The renaming problem extended** — the first analysis found that the same state variable appears under different names across genres. Does this pattern extend to the new primitive types? Do dynamics and goals have their own renaming problem?

### 2. Multi-Scale Structure Analysis

The dynamics and goals data uses a three-tier temporal model: orbital (bedrock), arc (sediment), scene (topsoil). The spatial topology data uses a directional flow model with friction and tonal inheritance.

- **How do these scale models interact?** An orbital dynamic constrains which arc dynamics are available. A spatial transition carries friction that maps to specific state variable costs. How should the data model represent these cross-system relationships?
- **Scale consistency across types** — do archetypes, goals, dynamics, and spatial topology use "scale" in compatible ways? Can we define a unified scale model, or do different primitive types need different temporal frameworks?
- **The geological metaphor as data structure** — bedrock/sediment/topsoil maps to persistence patterns (rarely changes / resistant to change / volatile). Does this map cleanly to database update frequencies and query patterns?

### 3. Relational Graph Edge Taxonomy

The dynamics data produces typed edges (debt-laden, parasitic, symbiotic, information-asymmetric, trust-textured). The archetype-dynamics data adds characteristic scenes and shadow pairings. The spatial topology adds friction and tonal inheritance edges.

- **What is the full edge taxonomy?** Read all dynamics, archetype-dynamics, and spatial-topology data. Catalog every distinct edge type, its properties, its directionality, and what it carries.
- **Edge properties as structured data** — which properties belong on every edge? Which are type-specific? Which are genre-specific?
- **Multi-scale edge properties** — the same edge between two characters may carry different state at the orbital, arc, and scene levels. How should this be represented?

### 4. Communicability as Queryable Structure

The place-entity data uses a four-channel communicability model (atmospheric, sensory, spatial, temporal). The ontological posture data determines which entities have communicability at all.

- **Can communicability be generalized beyond places?** Characters have communicability (how they express, what registers they speak in, what they withhold). Nonhuman agents have communicability (the Land communicates through weather and crop failure). Is the four-channel model a universal entity property or is it spatial-specific?
- **Communicability as query predicate** — the Storykeeper needs to ask "what can this character perceive about this place?" The answer depends on the place's communicability profile, the character's perceptual state, and the genre's epistemological stance. How should this be structured for efficient querying?

### 5. Narrative Shape Beat Structure

The narrative shapes data provides named beats with position, flexibility, tension contribution, and pacing function. These are the gravitational topology of the tilted sheet.

- **Beat compatibility** — can multiple narrative shapes be active simultaneously? (A character might be in the "Descent into Belonging" shape AND the "Ritual Countdown" shape.) How do overlapping shapes interact?
- **Beat recognition criteria** — the Dramaturge needs to recognize when the story has arrived at a new beat. What data from the narrative shapes should be structured as recognition criteria vs. as prose description?
- **Rest rhythms** — the narrative shapes include connector beat descriptions ("mundane observation as the land breathes," "Slow TV — deliberate, tactile, unhurried"). Should these be structured as a separate data type or kept as prose within the beat definitions?

### 6. Cross-Primitive Relationships

The primitive types are not independent. Archetypes have goals. Goals drive dynamics. Dynamics produce scenes (profiles). Scenes happen in places (place-entities). Places are connected by topology. Topology carries tonal inheritance. All of this is inflected by genre and ontological posture.

- **Map the dependency graph** between primitive types. Which types reference which other types? Are there circular dependencies?
- **Foreign key structure** — at the structured data level, how do records in different primitive types reference each other? By canonical name? By genre-specific variant name? By UUID?
- **The archetype-dynamics type as a cross-reference** — this type explicitly connects archetypes to dynamics to scenes. Is it the model for how all cross-references should work, or is it a special case?

### 7. JSON Schema Proposals

Based on all of the above, propose structured JSON schemas for each primitive type. These schemas will be used by a 7b-instruct model to extract structured records from the raw.md files.

For each schema:
- **Core fields** (always present)
- **Genre-variant fields** (present when the record is genre-specific)
- **Scale-qualified fields** (fields that carry a temporal scale: orbital/arc/scene)
- **Cross-references** (how this record points to records in other primitive types)
- **Flavor text fields** (qualitative descriptions preserved alongside structured values)

The schemas must serve the Storykeeper's query patterns:
- "Give me the archetype data for this character in this genre" → archetype record with genre variant
- "What dynamics are active for this character?" → graph traversal returning typed edges
- "What is the current narrative beat and what does this genre's rest rhythm look like?" → shape + beat records
- "What tonal inheritance does this place-entity send to adjacent places?" → spatial topology edge with inheritance profile

### 8. Gaps and Surprises

- **What's missing?** Read the full corpus and identify any structural gaps — places where the data implies a dimension or relationship that isn't explicitly captured by any primitive type.
- **What surprised you?** The data regularly produced insights that surprised the humans who generated it. Note anything that strikes you as unexpected, non-obvious, or particularly elegant.
- **Where does the data contradict itself?** Different genres may make incompatible claims about universal patterns. Note these tensions — they're often the most analytically valuable.

## Context You Should Know

- The engine uses PostgreSQL + Apache AGE for persistence (SQL tables + Cypher graph queries)
- The Storykeeper is a deterministic information boundary manager — bounded queries, structured results, never interprets
- The Dramaturge is an async agent that recognizes narrative position and issues dramatic directives
- Narrative events are gravity-based (accumulated graph state crosses thresholds) not trigger-based (authored conditions)
- The narrative graph has directional flow — downstream for gravitational pull, upstream only through memory
- Genre is a dynamic transform, not a static initial condition — it responds to state variable changes
- Characters are composed of one primary archetype + one shadow archetype, with personal variance
- The generated data is a *translation layer* for how intent becomes meaning, not a rules/constraint system

## How to Work

Read the full corpus systematically. Don't sample — read everything. The patterns that matter most are the ones that only appear when cross-referencing across types and across genres. Use sub-agents freely for parallel reading if that helps manage the volume.

Start with the genre regions (they're the foundation everything else builds on). Then read the cluster syntheses for each primitive type (they're the most condensed form of the cross-genre patterns). Then dive into per-genre extractions for specific examples that illustrate or complicate the cluster-level patterns.

The commentary and suggestions sections in the raw.md files are often the most analytically valuable — they contain the model's own reflections on patterns, tensions, and surprises.
